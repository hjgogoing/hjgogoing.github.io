字段类型

1.整数 int
2.长整型 long
3.单精度 float
4.双精度 double
5.用来保存具有小数点而且数值确定的数 Decimal 一般涉及到金钱的可以用此变量。
6.字符串
char 字节 定长0-255长度 jepsonxxxxxx 自动补齐
varchar 字符串 变长0-65535字节 jepson
7.日期
date 日期 YYYY-MM-DD
time 时间 HH:MM:SS
datetime 年月日时分秒
timestamp 年月日时分秒
小知识点：datetime 与 timestamp的区别参考连接
1)储存方式不同：
对于TIMESTAMP，它把客户端插入的时间从当前时区转化为UTC（世界标准时间）进行存储。查询时，将其又转化为客户端当前时区进行返回。而对于DATETIME，不做任何改变，基本上是原样输入和输出
2两者所能存储的时间范围不一样
timestamp所能存储的时间范围为：’1970-01-01 00:00:01.000000’ 到 ‘2038-01-19 03:14:07.999999’。
datetime所能存储的时间范围为：’1000-01-01 00:00:00.000000’ 到 ‘9999-12-31 23:59:59.999999’。

sql类型

1.ddl 数据定义语言 例如：create drop
2.dml 数据操作语言 例如：select insert update delete 增删改查
3.dcl 数据控制语言 例如：grant

建表规范

规范说明：
1.表名称 字段名称 不要写中文 尽量不要汉语拼音。

2.统一风格:在现存的项目中，遵循已有的风格，新项目要设计好风格。

3.第一个字段必须是自增长字段 主键，且 无业务意义 架构设计默认规则。

4.一张表只有一个主键 id，业务字段需要唯一的话，就使用唯一约束来保证。
ALTER TABLE ruozedata.rz ADD CONSTRAINT rz_un UNIQUE KEY (name);
