查看磁盘信息

磁盘一般分为系统盘和数据盘，系统盘挂载在/根目录，一般可以设置100G。数据盘挂载在/data01这种形式的目录下。

参看内存信息

样例：
主要看tota与used。通过used/tota可以得到内存的使用率。一般需要预留15%的内存。swap 因为内存不够，使用部分磁盘空间来充当内存使用，虽然可以解决内存紧缺的问题，但是效率不高。尤其大数据领域，swap哪怕设置了大小 ，也尽量设置惰性使用。

出现负载过高的情况可以从以下两个方面考虑：
a.计算程序 hive sql、spark 、flink 密集计算 是不是要调优
b.是不是被挖矿了
c.硬件问题，内存条损坏。最后一招，万能重启，检测是不是硬件问题

显示网络状态

样例1
tcp6 0 0 :::80 :::* LISTEN 3478/httpd
tcp6 0 0 0.0.0.0:80 :::* LISTEN 3478/httpd
tcp6 0 0 192.168.1.101:80 :::* LISTEN 3478/httpd
80端口号服务可以对外
window或者其他服务器 可以ip+80 进行访问的
样例2
tcp6 0 0 localhost:80 :::* LISTEN 3478/httpd
tcp6 0 0 127.0.0.1:80 :::* LISTEN 3478/httpd
localhost 、127.0.0.1 代表本机
80端口仅限本机的其他服务可以访问，window或者其他服务器无法访问

小结

a.有进程PID 不一定就有端口号
b.服务的通信交流，其实就是要 ip+端口号
c.可能出现的错误：Trying 114.67.101.143… Connection refused
d.相关问题：那个机器上运行xxx服务，把那个web打开一下 ？
解决办法:
1.已知IP地址
2.ps -ef | grep xxx 找到pid
3.netstat -nlp| grep pid 找到对应端口号
4.window: 提前安装telnet客户端。linux : yum install -y telnet
5.先 ping ip再 telnet ip port
